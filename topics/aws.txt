VPC Amazon Virtual Private Cloud - логічно ізольований розділ хмари амазон, в якому можна запускати ресурси aws в створеній користувачем відртуальній мережі (діапазони адрес, підмережі, шлюзи, таблиці маршрутизації), можна навіть піключити через vpn  до корпоративної залізної мережі для її розширення

VPC peering - мережеве з"єднання між двома VPC, інстанси комунікують так, ніби знаходяться в одному VPC. Можна настроювати навіть між різними AWS регіонами

AMI - AMI — Amazon Machine Image - можливість створити свій образ операційної системи

Route53 - сервіс системи доменних імен (DNS), працює з ip4 i ip6, при покупці домена і керуванні ними Route53 автоматично налаштує параметри dns

S3 - об"єктне сховище, призначене для зберігання будь-яких об"єктів

EBS - Elastic Block Store - волюми для підключення до інстансів, на них потіи иожна створювати файлові системи, їх можна реплікувати Elastic Volumes - можливість дмнамічно змінювати розмір EBS без впливу на роботу
	типи томів
		ssd:
			-io1 - самі швидкі
			- gp2 - універсальні
		hdd:
			-st1 - з оптимізованою пропускною здатністю
			-sс1 - "холодний", для робочих авантажень з нечастим доступом

ELB Elastic Load Balancing - автоматично розподіляє вхідний трафік по різних об"єктах (інстанси, контейнери, ір-адреси)в одній зоні або декількох
	типи:
	-Application Load Balancer: найкраще для балансування трафіка http i https , працює на 7 рівні(направляє трафік на ціль, залежно від вмісту запита)
	-Network Load Balancer: найкраще для балансування трафіка TCP (4 рівень), для раптового і змінюваного навантаження
	-Classic Load Balancer: базове балансування навантаження між інстансамм (працює як на рівні запита так і на рівні з"єднання)

SES Simple Email Service - хмарний сервіс відправки електронної пошти

BIND (Berkeley Internet Name Domain) - відкрита і найбільш розповсюджена реалізація dns-сервера (10 з 13 кореневих dns серверів працюють на bind)

Кожен регіон поділений на декілька зон доступності (фізично відокремлених і з"єднаних високошвидкісними лініями зв"язку) 55 зон в 12 регіонах

типи dns записів
	А - зв"язок домена і ір-адреси
		site.ru		А 123.123.123.123
	CNAME (Canonical name) - в основоному для переадресації піддомена на інший домен
		webmail.site.ru	CNAME	webmail.hosting.reg.ru
	MX (Mail Exchanger) - адреса поштового шлюза для домена (ім"я, тип, пріорітет, значення)
		site.ru	MX	10	mx1.hosting.reg.ru
	NS - алреса вузла, який ідповідає за доменну зону (авторитативного)
	PTR - зворотний запис (зв"язує ір з іменем) - часто використовується при фільтрації пошти
	SOA (Start of Authority) - вказує на якому сервері зберігається еталонна інфориація про доменне ім"я
	SPF (Sender Policy Framework) - сервера, які можуть відправляти пошту від імені домена
	TXT (Text string)  - текстовий запис, для перевірок на право володіння доменом і для SPF і ключа DKIM
