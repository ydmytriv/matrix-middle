nosql - нереляційні БД
	- можуть зберігати дані як одне ціле (наприклад 1 json)
	- кожне рішення використовує свій формат запитів
	- немає однозначно визначеної структури зберігання даних
	- зберігають документи замість строк, які можуть зберігти складну по структурі інформацію
	- легше горизонтально масштабуються 
	- менша надійність ніж в SQL
	- важче отримати підтримку
	- добре підходить для випадків, коли не використовуються залежності і потрібна масштабованість

Mongo - документоорієнтована ДБ (пошук по полях наприклад в великому джейсоні)
	використовується наприклад для блог-платформи або каталога продуктів

Neo4j - графова БД (вузли - суб"єкти, грані - зв"язки)
	наприклад для соц.мережі, дуже добре справляється з створенням дуже глибоких зв"язків

Redis (кеш) - коли потрібен дуже швидкий доступ до даних
	наприклад при інтернет торгівлі, дані, до яких часто звертаються зберігаються в кеші і не потрібно робити запити до ДБ

ElasticSearch(пошукові) - використовуються коли потрібно робити пошук по великому масиву, повнотекстовий пошук з широким функціоналом

Cassandra(строкові БД) - для зберігання послідовних даних, логів, чи іншої генерованої інформації, наприклад від датчиків
	мало запитів на читання, в основному запис

Реплікація - повна копія БД

Шардінг - БД по різних серверах
	Вертикальний шардінг - різні таблиці на різних серверах
	Горизонтальний шардінг - частина великої таблиці на одному сервері, інша - на іншому

формати даних
	JSON (JavaScript Object Notation) - ефективно описує складні по структурі дані
	MongoDB - bson (binary json) - набагато швидший пошук по ньому але займає більше місця
